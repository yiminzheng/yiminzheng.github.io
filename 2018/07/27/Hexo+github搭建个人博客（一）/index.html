Hexo+github搭建个人博客（一）

发表于 2018-07-27 | 分类于 博客搭建

环境准备

  1. 安装Node。

    可下载到桌面默认路径安装即可。
    在cmd中输入:node -v,检验node是否安装正确。

    $node -v
    v8.9.4

    本人安装的v8.9.4版本。出现上述内容安装正确。

  1. 安装Git。

    可下载到桌面默认路径安装即可。在桌面或文件夹空白处鼠标右键菜单会新增“Git GUI Here”和“Git Bash Here”两个选项。
    在git bash中输入:git --version,检验git是否安装正确。

    $git –version
    git version 2.17.0.windows.1

    出现上述内容即为安装正确。

  • 创建GitHub Pages并配置ssh

    • 创建博客仓库
      注册并登陆到GitHub,根据自己的用户名新建仓库，举例：xxxx.github.io
    • 本地生成ssh密钥
      git bash下输入ssh-keygen -t rsa -C '你的github邮箱地址'
    • 上传本地公钥串，使当前电脑与GitHub账户建立联系。

  1. 选装cnmp。若可翻墙即直接安装nmp。选装cnmp时后面使用时将命令中nmp换成cnmp即可。

    在cmd中输入命令

    $npm install -g cnpm –registry=https://registry.npm.taobao.org

  2. 安装Hexo及相关

  • 全局安装Hexo
    在桌面空白处点击右键，若不是默认安装就要选择安装路径在安装文件中打开，打开git bash，输入如下：

    $nmp install hexo-cli -g

    也可以在cmd中输入，如果在cmd中出现npm不是内部外部命令等，则说明不是全局变量。

    安装完成后输入如下：

    hexo -v
    hexo-cli: 1.1.0
    os: Windows_NT 6.1.7601 win32 x64..

    版本信息挺多，出现则表示安装成功。

  • 在项目中安装Hexo
    选择合适路径建立blog文件夹并进入，在该文件夹下打开git bash，依次输入：

    $npm install hexo –save #在当前项目中安装hexo；
    hexo init #初始化hexo；
    npm install #安装依赖包；

  • 安装hexo的插件

    npm install hexo-server –save #搭建本地服务器所需插件
    npm install hexo-deployer-git –save #使用git方式进行部署博客所需插件

在本地生成博客静态页面并预览

  • hexo generate:简写hexo g，会生成一个存放静态文件的文件夹public；
  • hexo server：简写hexo s，启动本地服务器
  • 默认本地网址：http://localhost:4000/
    会看见一个Hexo博客初始化页面。但是这些只能在本机看见，要想让别人也能通过域名访问，我们采取将内容推到github上。
  • 效果图

本地博客同步到GitHub上

  • 即将blog文件夹中./public文件夹内容推送到github仓库xxxx.github.io中。

  1. 编辑博客配置文件：_config.yml
    在blog文件夹下找到该配置文件，将其中参数deploy按如下格式更新，冒号：一定要有空格

    deploy:
    type: git
    repo: git@github.com:xxxx/xxxx.github.io.git
    branch: master

  2. 重新部署
    在博客根目录下打开git bash,依次执行Hexo命令：

    hexo clean #会清除缓存文件db.json及之前生成的静态文件夹public；
    hexo g #会重新生成静态文件夹public；
    hexo deploy #会在博客根目录下生成一个.deploy_git的文件夹，并 把本地生成的静态文件部署到xxxx.github.io这个仓库中的master分支上；简写形式为hexo d；

    hexo g和hexo d可以合并成hexo g -d;

  3. 在浏览器中访问博客
    在浏览器输入xxxx.github.io

至此，我们就通过Hexo创建了一个原始的博客，并通过把博客静态文件放到GitHub仓库中，可以通过访问github仓库访问我们的博客。下面可以对博客的主题域名等进行进一步的优化。
